§ 00 · A thought facility

Meeting transcripts that actually ship something.

Weekly conversations of the 8am-AI peer group, walked through a skills pipeline that mines themes and walks promising threads to published artifacts — blogs, gists, runnable experiments. Local-first. Installs as a Claude Code plugin.

Install in two lines View on GitHub
§ 01 · Why this exists

Peer groups produce a lot of signal and very few durable artifacts.

8am-ai is an experiment in fixing that — by treating the transcript as the source of truth and walking each surfaced thought through a small, opinionated pipeline that ends in something shareable. If it works, the corpus becomes a book. If it doesn't, we publish what we learned about why.

§ 03 · Pipeline

From transcript to artifact in four moves.

Each step is a single skill. Phase is the queue — anything not yet "published" is in the orchestrator's pile.

01 · Intake

Harvest & normalize

Pulls transcripts from Google Drive (or local drop). Normalizes Fireflies JSON, summaries, DOCX into utterance windows.

02 · Mine

Classify & extract

Ideas, directives, mentions, curiosities — every record cites the source window so it stays traceable.

03 · Synthesize

Cluster & promote

Longitudinal theme detection across weeks. Promising ideas get assigned a track — light (blog) or heavy (experiment).

04 · Ship

Publish & announce

Secret-scrubbed gist + scsiwyg blog (editorial, unlisted) + optional Slack ping. Null results are still publishable.

§ 04 · Tracks

Not every thought wants to be a runnable experiment.

Two tracks come off the same intake. The synthesizer assigns based on whether the idea proposes a buildable artifact or a measurable claim.

Light · essay track

For ideas that want to be argued, not built. Researched, drafted, and shipped as a blog post + gist.

extracted→classified→researched→drafted→published

Heavy · experiment track

For claims that can be tested. Bounded experiment runs in a sandbox with a kill criterion and a budget. Failed experiments still ship.

extracted→classified→researched→experimented→published
§ 05 · Skills

Eight slash commands, one shared substrate.

All reads/writes route through meetup-state — the single writer. The rest are workers.

/meetup-state scaffoldSubstrate spine. The only skill permitted to write ideas, themes, directives, and state to disk.
/meetup-harvester-gdrive scaffoldPull new transcripts from the shared Google Drive folder; advance per-surface cursor.
/meetup-intake scaffoldNormalize transcripts (Fireflies JSON / MD / DOCX) into utterance windows.
/meetup-classify scaffoldMine ideas, directives, mentions, curiosities. Every record cites a source window.
/meetup-synthesizer scaffoldCluster ideas across weeks → THEME records. Assign light or heavy track.
/meetup-experimenter scaffoldHeavy track: bounded runnable test in a sandbox; outcome.yaml records hypothesis result.
/meetup-publisher scaffoldSecret-scrubbed gist + scsiwyg blog post (editorial theme, unlisted by default).
/meetup-orchestrator scaffoldWeekly walk — thin router over the lifecycle DAG; honors weekly budget; carries unfinished work forward.
